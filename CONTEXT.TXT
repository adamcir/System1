Implemented modules in all branches:
- vga
- bootlog
- panic
- interrupts (full IDT/PIC/IRQ implementation)
- keyboard (IRQ1 + optional polling fallback)

VGA status:
- Default color: LIGHT_GREY
- API:
  - vga_set_color(...)
  - vga_set_cursor(row, col)
  - vga_get_cursor(&row, &col)
  - vga_putc_at(row, col, c)
- Hardware text cursor is synchronized (port 0x3D4/0x3D5).
- Enter/newline cursor movement fixed.

Keyboard status:
- IRQ1 handler reads scancodes from ports 0x64/0x60.
- Polling path is optional fallback (disabled by default).
- Supports regular ASCII keys + Shift.
- Supports Backspace.
- Supports arrow Left and Right.
- Arrow Up and Down intentionally ignored.
- New API:
  - keyboard_take_key()
  - keyboard_take_char()

Kernel loop behavior:
- Calls keyboard_poll() only as optional fallback path.
- Echoes typed chars to VGA.
- Backspace erases previous character on screen.
- Left/Right arrows move cursor.

Interrupts status:
- PIC remap active: master 0x20, slave 0x28.
- IRQ EOI handling implemented for master/slave.
- PIT IRQ0 configured and timer ticks increment in IRQ handler.
- `irq_register_handler(...)` now also updates PIC masks.
- Slave IRQ enable path correctly handles cascade IRQ2.
- `timer_ticks_get()` uses interrupt-safe read section.
- IDT gate selector now uses runtime `CS` value (not hardcoded 0x08).

Boot/runtime validation:
- `make clean && make iso-32 iso-64 img-32` passes.
- `iso-32` in QEMU `-nographic` no longer falls into reboot loop.
- Root cause of previous reboot loop was triple fault after first timer IRQ:
  - IDT entries used wrong selector (`0x08`) while runtime `CS` was `0x10`.
  - Fixed by setting selector from current `CS` during `interrupts_init()`.

Floppy stability fixes already in place:
- stage1/stage2 LBA->CHS register clobber fix.
- linker.floppy.i386.ld base aligned to 0x00010000.

Known warnings:
- Linker warns about RWX LOAD segments (accepted in current MVP).

Next technical priority:
- Run equivalent runtime validation for `iso-64` and `img-32` paths.
